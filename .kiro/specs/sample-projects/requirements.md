# Requirements Document

## Introduction

This feature adds a top-level `samples/` directory containing two self-contained sample projects — one using Maven and one using Gradle — that demonstrate how to consume the published Rawit annotation processor (`io.github.projectrawit:rawit:0.0.3`). Each sample showcases the `@Invoker` and `@Constructor` annotations, the generated fluent API, and includes JUnit 5 tests to verify correctness.

## Glossary

- **Maven_Sample**: The sample project located at `samples/maven-sample/` that uses Apache Maven as its build tool
- **Gradle_Sample**: The sample project located at `samples/gradle-sample/` that uses Gradle (Groovy DSL) as its build tool
- **Rawit_Dependency**: The published Maven Central artifact `io.github.projectrawit:rawit:0.0.3`
- **Two_Pass_Compile**: A build configuration where the first pass compiles sources without annotation processing (`-proc:none`) and the second pass runs annotation processing only (`-proc:only`) against the already-compiled `.class` files
- **Staged_Call_Chain**: The fluent API generated by Rawit where each parameter becomes a chained method call, ending with `.invoke()` or `.construct()`
- **Invoker_Annotation**: The `@Invoker` annotation placed on non-private methods to generate a parameterless overload returning a staged call chain
- **Constructor_Annotation**: The `@Constructor` annotation placed on non-private constructors to generate a `public static constructor()` method returning a staged construction chain
- **Sample_Test_Suite**: The JUnit 5 test classes within each sample project that verify the generated fluent API works correctly

## Requirements

### Requirement 1: Maven Sample Project Structure

**User Story:** As a developer evaluating Rawit, I want a complete Maven sample project, so that I can see how to integrate Rawit into a Maven build.

#### Acceptance Criteria

1. THE Maven_Sample SHALL contain a valid `pom.xml` at `samples/maven-sample/pom.xml`
2. THE Maven_Sample SHALL declare a dependency on Rawit_Dependency with groupId `io.github.projectrawit`, artifactId `rawit`, and version `0.0.3`
3. THE Maven_Sample SHALL configure the `maven-compiler-plugin` with Two_Pass_Compile: a `default-compile` execution using `-proc:none` and a `process-annotations` execution in the `process-classes` phase using `-proc:only`
4. THE Maven_Sample SHALL target Java 17 as the minimum language level
5. THE Maven_Sample SHALL include JUnit 5 as a test dependency

### Requirement 2: Gradle Sample Project Structure

**User Story:** As a developer evaluating Rawit, I want a complete Gradle sample project, so that I can see how to integrate Rawit into a Gradle build.

#### Acceptance Criteria

1. THE Gradle_Sample SHALL contain a valid `build.gradle` at `samples/gradle-sample/build.gradle`
2. THE Gradle_Sample SHALL declare Rawit_Dependency as both an `annotationProcessor` and `compileOnly` dependency with groupId `io.github.projectrawit`, artifactId `rawit`, and version `0.0.3`
3. THE Gradle_Sample SHALL configure Two_Pass_Compile: a `compileJava` task with `-proc:none` and a `processAnnotations` task of type `JavaCompile` with `-proc:only` that depends on `compileJava`
4. THE Gradle_Sample SHALL wire the `classes` task to depend on the `processAnnotations` task
5. THE Gradle_Sample SHALL target Java 17 as the minimum language level
6. THE Gradle_Sample SHALL include JUnit 5 as a test dependency with `useJUnitPlatform()` configured

### Requirement 3: Demonstrate @Invoker on Instance Methods

**User Story:** As a developer, I want to see @Invoker used on an instance method, so that I can understand how to generate a staged call chain for instance methods.

#### Acceptance Criteria

1. THE Maven_Sample SHALL contain a class with at least one non-private instance method annotated with Invoker_Annotation that has at least two parameters
2. THE Gradle_Sample SHALL contain a class with at least one non-private instance method annotated with Invoker_Annotation that has at least two parameters
3. WHEN the Maven_Sample is compiled with Two_Pass_Compile, THE Rawit_Dependency SHALL generate a parameterless overload on the instance method that returns a Staged_Call_Chain ending with `.invoke()`
4. WHEN the Gradle_Sample is compiled with Two_Pass_Compile, THE Rawit_Dependency SHALL generate a parameterless overload on the instance method that returns a Staged_Call_Chain ending with `.invoke()`

### Requirement 4: Demonstrate @Invoker on Static Methods

**User Story:** As a developer, I want to see @Invoker used on a static method, so that I can understand how staged call chains work for static methods.

#### Acceptance Criteria

1. THE Maven_Sample SHALL contain a class with at least one non-private static method annotated with Invoker_Annotation that has at least two parameters
2. THE Gradle_Sample SHALL contain a class with at least one non-private static method annotated with Invoker_Annotation that has at least two parameters
3. WHEN the Maven_Sample is compiled with Two_Pass_Compile, THE Rawit_Dependency SHALL generate a parameterless static overload that returns a Staged_Call_Chain ending with `.invoke()`
4. WHEN the Gradle_Sample is compiled with Two_Pass_Compile, THE Rawit_Dependency SHALL generate a parameterless static overload that returns a Staged_Call_Chain ending with `.invoke()`

### Requirement 5: Demonstrate @Constructor

**User Story:** As a developer, I want to see @Constructor used on a constructor, so that I can understand how to generate a staged construction chain.

#### Acceptance Criteria

1. THE Maven_Sample SHALL contain a class with a non-private constructor annotated with Constructor_Annotation that has at least two parameters
2. THE Gradle_Sample SHALL contain a class with a non-private constructor annotated with Constructor_Annotation that has at least two parameters
3. WHEN the Maven_Sample is compiled with Two_Pass_Compile, THE Rawit_Dependency SHALL generate a `public static constructor()` method that returns a Staged_Call_Chain ending with `.construct()`
4. WHEN the Gradle_Sample is compiled with Two_Pass_Compile, THE Rawit_Dependency SHALL generate a `public static constructor()` method that returns a Staged_Call_Chain ending with `.construct()`

### Requirement 6: Demonstrate Usage of Generated Fluent API

**User Story:** As a developer, I want to see the generated fluent API being called, so that I can understand the usage pattern for staged call chains.

#### Acceptance Criteria

1. THE Maven_Sample SHALL contain example code that invokes an instance method via the Staged_Call_Chain pattern: `instance.method().param1(value1).param2(value2).invoke()`
2. THE Maven_Sample SHALL contain example code that invokes a static method via the Staged_Call_Chain pattern: `ClassName.method().param1(value1).param2(value2).invoke()`
3. THE Maven_Sample SHALL contain example code that constructs an object via the Staged_Call_Chain pattern: `ClassName.constructor().param1(value1).param2(value2).construct()`
4. THE Gradle_Sample SHALL contain example code that invokes an instance method via the Staged_Call_Chain pattern: `instance.method().param1(value1).param2(value2).invoke()`
5. THE Gradle_Sample SHALL contain example code that invokes a static method via the Staged_Call_Chain pattern: `ClassName.method().param1(value1).param2(value2).invoke()`
6. THE Gradle_Sample SHALL contain example code that constructs an object via the Staged_Call_Chain pattern: `ClassName.constructor().param1(value1).param2(value2).construct()`

### Requirement 7: JUnit 5 Tests for Generated API

**User Story:** As a developer, I want unit tests in each sample project that verify the generated API, so that I can confirm the annotation processor works end-to-end.

#### Acceptance Criteria

1. THE Maven_Sample SHALL contain a JUnit 5 test class in `src/test/java` that verifies the instance method Staged_Call_Chain returns the correct result
2. THE Maven_Sample SHALL contain a JUnit 5 test class in `src/test/java` that verifies the static method Staged_Call_Chain returns the correct result
3. THE Maven_Sample SHALL contain a JUnit 5 test class in `src/test/java` that verifies the constructor Staged_Call_Chain produces an object with the correct field values
4. THE Gradle_Sample SHALL contain a JUnit 5 test class in `src/test/java` that verifies the instance method Staged_Call_Chain returns the correct result
5. THE Gradle_Sample SHALL contain a JUnit 5 test class in `src/test/java` that verifies the static method Staged_Call_Chain returns the correct result
6. THE Gradle_Sample SHALL contain a JUnit 5 test class in `src/test/java` that verifies the constructor Staged_Call_Chain produces an object with the correct field values

### Requirement 8: Sample Projects Are Self-Contained

**User Story:** As a developer, I want each sample project to be independently buildable, so that I can clone and run either sample without needing the parent project.

#### Acceptance Criteria

1. THE Maven_Sample SHALL be buildable by running `mvn clean test` from the `samples/maven-sample/` directory without requiring any parent POM or local repository setup beyond Maven Central
2. THE Gradle_Sample SHALL be buildable by running `gradle clean test` from the `samples/gradle-sample/` directory without requiring any parent build configuration beyond Maven Central
3. THE Maven_Sample SHALL resolve Rawit_Dependency from Maven Central
4. THE Gradle_Sample SHALL resolve Rawit_Dependency from Maven Central via the `mavenCentral()` repository
